The SQL DSL

The SQL dsl attempts to closely mimic standard SQL syntax.

To get started with the DSL, import the scoobie DSL, and an interpreter.

scala import scoobie.doobie.doo.postgres._ import scoobie.snacks.mild.sql._

From here, you can start throwing queries together.

scala val simpleSelect = select(p"foo", p"baz") from p"bar" // simpleSelect: scoobie.snacks.mild.sql.QueryBuilder[scoobie.doobie.ScoobieFragmentProducer] = QueryBuilder(QueryProjectOne(QueryPathEnd(bar),None),List(QueryProjectOne(QueryPathEnd(foo),None), QueryProjectOne(QueryPathEnd(baz),None)),List(),QueryComparisonNop,List(),List(),None,None)

The above expression wil generate a QueryBuilder for a simple select query. You can attach more to it with things like where, offset, limit, orderBy, leftOuterJoin, and more. Before we get there, lets talk about p.

Paths

The p string interpolator will take a standard string, and convert it to a QueryPath for use in queries, seen below

```scala val shortPath = p”foo” // shortPath: scoobie.ast.QueryPath[scoobie.doobie.ScoobieFragmentProducer] = QueryPathEnd(foo)

val longPath = p”foo.bar” // longPath: scoobie.ast.QueryPath[scoobie.doobie.ScoobieFragmentProducer] = QueryPathCons(foo,QueryPathEnd(bar))

val longerPath = p”foo.bar.baz” // longerPath: scoobie.ast.QueryPath[scoobie.doobie.ScoobieFragmentProducer] = QueryPathCons(foo,QueryPathCons(bar,QueryPathEnd(baz))) ```

The differences between using a QueryPath and a String in a query are pretty drastic. Consider these two queries

scala val compareColumn = select(p"foo") from(p"bar") where p"foo" === p"baz" val compareValue = select(p"foo") from(p"bar") where p"foo" === "baz"

The compareColumn query will compare columns foo and baz, while the compareValue query will compare the value in column foo to the string “baz”.

Where

Where clauses are constructed using mostly familiar scala comparisons:

```scala val gt = p”x” > 30 // gt: scoobie.ast.QueryComparison[scoobie.doobie.ScoobieFragmentProducer] = QueryGreaterThan(QueryPathEnd(x),QueryParameter(30))

val lt = p”x” < p”y” // lt: scoobie.ast.QueryComparison[scoobie.doobie.ScoobieFragmentProducer] = QueryLessThan(QueryPathEnd(x),QueryPathEnd(y))

val gte = p”x” >= 30 // gte: scoobie.ast.QueryComparison[scoobie.doobie.ScoobieFragmentProducer] = QueryGreaterThanOrEqual(QueryPathEnd(x),QueryParameter(30))

val lte = p”x” <= p”y” // lte: scoobie.ast.QueryComparison[scoobie.doobie.ScoobieFragmentProducer] = QueryLessThanOrEqual(QueryPathEnd(x),QueryPathEnd(y))

val eq = p”a” === false // eq: scoobie.ast.QueryComparison[scoobie.doobie.ScoobieFragmentProducer] = QueryEqual(QueryPathEnd(a),QueryParameter(false))

val oneOf = p”a” in (“foo”, “bar”, “baz”) // oneOf: scoobie.ast.QueryComparison[scoobie.doobie.ScoobieFragmentProducer] = QueryIn(QueryPathEnd(a),List(QueryParameter(foo), QueryParameter(bar), QueryParameter(baz)))

val notOneOf = p”a” notIn (“foo”, “bar”, “baz”) // not (p”foo” in …) works as well. // notOneOf: scoobie.ast.QueryComparison[scoobie.doobie.ScoobieFragmentProducer] = QueryNot(QueryIn(QueryPathEnd(a),List(QueryParameter(foo), QueryParameter(bar), QueryParameter(baz))))

val invert = not(p”a” >= 10) // invert: scoobie.ast.QueryNot[scoobie.doobie.ScoobieFragmentProducer] = QueryNot(QueryGreaterThanOrEqual(QueryPathEnd(a),QueryParameter(10))) ```

Start a where clause in a traditional where position, like during a join, select, update, or delete.

```scala val selectWhere = select(p”foo”) from(p”bar”) where (p”bar.id” === 3) // selectWhere: scoobie.snacks.mild.sql.QueryBuilder[scoobie.doobie.ScoobieFragmentProducer] = QueryBuilder(QueryProjectOne(QueryPathEnd(bar),None),List(QueryProjectOne(QueryPathEnd(foo),None)),List(),QueryAnd(QueryComparisonNop,QueryEqual(QueryPathCons(bar,QueryPathEnd(id)),QueryParameter(3))),List(),List(),None,None)

val updateWhere = update(p”bar”) set (p”foo” ==> “baz”) where (p”bar.id” === 3) // updateWhere: scoobie.snacks.mild.sql.UpdateBuilder[scoobie.doobie.ScoobieFragmentProducer] = UpdateBuilder(QueryPathEnd(bar),List(ModifyField(QueryPathEnd(foo),QueryParameter(baz))),QueryAnd(QueryComparisonNop,QueryEqual(QueryPathCons(bar,QueryPathEnd(id)),QueryParameter(3))))

val deleteWhere = deleteFrom(p”foo”) where(p”bar.id” === 3) // deleteWhere: scoobie.ast.QueryDelete[scoobie.doobie.ScoobieFragmentProducer] = QueryDelete(QueryPathEnd(foo),QueryEqual(QueryPathCons(bar,QueryPathEnd(id)),QueryParameter(3)))

val joinOn = select(p”foo”) from(p”bar”) leftOuterJoin((p”baz” as “baz”) on (p”bar.id” === p”baz.bar_id”)) // joinOn: scoobie.snacks.mild.sql.QueryBuilder[scoobie.doobie.ScoobieFragmentProducer] = QueryBuilder(QueryProjectOne(QueryPathEnd(bar),None),List(QueryProjectOne(QueryPathEnd(foo),None)),List(QueryLeftOuterJoin(QueryProjectOne(QueryPathEnd(baz),Some(baz)),QueryEqual(QueryPathCons(bar,QueryPathEnd(id)),QueryPathCons(baz,QueryPathEnd(bar_id))))),QueryComparisonNop,List(),List(),None,None) ```

You can combine multiple comparisons using and/or.

scala val foo = p"x" > 30 and p"y" < 10 // foo: scoobie.ast.QueryAnd[scoobie.doobie.ScoobieFragmentProducer] = QueryAnd(QueryGreaterThan(QueryPathEnd(x),QueryParameter(30)),QueryLessThan(QueryPathEnd(y),QueryParameter(10)))

Joins

Using the DSL, you create join clauses just as you would in sql (albeit with a bit more syntax)

scala select(p"a.foo", p"b.bar") from (p"table1" as "a") leftOuterJoin ((p"table2" as "b") on (p"a.id" === p"b.a_id")) // res0: scoobie.snacks.mild.sql.QueryBuilder[scoobie.doobie.ScoobieFragmentProducer] = QueryBuilder(QueryProjectOne(QueryPathEnd(table1),Some(a)),List(QueryProjectOne(QueryPathCons(a,QueryPathEnd(foo)),None), QueryProjectOne(QueryPathCons(b,QueryPathEnd(bar)),None)),List(QueryLeftOuterJoin(QueryProjectOne(QueryPathEnd(table2),Some(b)),QueryEqual(QueryPathCons(a,QueryPathEnd(id)),QueryPathCons(b,QueryPathEnd(a_id))))),QueryComparisonNop,List(),List(),None,None)

Valid joins include leftOuterJoin, rightOuterJoin, innerJoin, fullOuterJoin, and crossJoin`.
